# 课程项目要求说明

## 一、项目选题

实现任意一种**管理系统**，例如：

* 订单管理系统
* 学生信息管理系统

---

## 二、中间件使用要求

项目中**至少必须使用以下中间件**：

* 数据库（类型不限）
* Kafka 消息中间件

---

## 三、前端实现要求（20 分）

### 3.1 前端开发

| 要求项                   | 分值   |
| --------------------- | ---- |
| 有前端界面                 | 6 分  |
| 使用前端模板模式（如 Thymeleaf） | 8 分  |
| 前后端分离模式（如 Vue）        | 10 分 |

---

### 3.2 前端展示

| 要求项      | 分值   |
| -------- | ---- |
| 单层页面跳转   | 6 分  |
| 多层界面逻辑跳转 | 7 分  |
| 图片展示     | 8 分  |
| CSS 样式   | 10 分 |

---

## 四、安全管理（20 分）

### 4.1 用户登录

| 实现方式           | 分值   |
| -------------- | ---- |
| 基于内存的用户登录      | 6 分  |
| 使用密码加密         | 7 分  |
| 基于数据库的用户登录     | 8 分  |
| 基于 Redis 的用户登录 | 10 分 |

---

### 4.2 权限控制

| 实现方式              | 分值   |
| ----------------- | ---- |
| 有权限控制             | 6 分  |
| 基于角色的权限控制         | 8 分  |
| 基于方法的权限控制         | 9 分  |
| 权限持久化存储在数据库或其他中间件 | 10 分 |

---

## 五、基本功能要求（30 分）

### 5.1 重要接口调用统计（10 分）

实现对系统中重要接口调用次数的统计：

| 实现方式                  | 分值   |
| --------------------- | ---- |
| 使用 AOP + 内存           | 7 分  |
| 使用 AOP + Redis        | 8 分  |
| 使用 AOP + Redis + 前端展示 | 10 分 |

---

### 5.2 数据持久化（10 分）

| 要求                                    | 分值   |
| ------------------------------------- | ---- |
| 使用 Spring Data JPA 或 Spring Data JDBC | 8 分  |
| 知晓如何切换持久层框架（如 MyBatis）                | 10 分 |

---

### 5.3 Kafka 消息机制（10 分）

* 使用 Kafka 对消息进行生产与消费

---

### 5.4 Redis 缓存使用（10 分）

* 使用 Redis 缓存
* 使用 Redis 的不同数据结构（如 `set`、`zset`、`list` 等）解决特定业务场景问题

---

## 六、部署要求（15 分）

| 部署方式                 | 分值   |
| -------------------- | ---- |
| 本地部署                 | 8 分  |
| 使用 Dockerfile 部署     | 12 分 |
| 使用 docker-compose 部署 | 15 分 |
| Kubernetes 部署（附加分）   | +5 分 |

---

## 七、文档要求（15 分）

* 撰写一篇 **2000 字以内**
* 内容为 **Spring Boot 学习总结**

---

## 八、说明

* 项目需覆盖上述各模块要求
* 不同实现方式对应不同评分等级
* 鼓励采用更高阶技术方案以获取更高分数

---
